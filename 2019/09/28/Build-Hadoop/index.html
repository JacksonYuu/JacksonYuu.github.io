There are three environments for building Hadoop: single-machine mode, pseudo-distributed and fully distributed. This paper introduces how to build a fully distributed Hadoop environment with virtual machines. There are mainly NameNode and Econdary NameNode in the same node, DataNode in three nodes.

(搭建Hadoop的环境有三种：单机模式，伪分布式，全分布式。在这里介绍的是用虚拟机去搭建全分布式的Hadoop环境。主要有NameNode和SecondaryNameNode在同一个节点，DataNode分别在三个节点。)

3、Third Step（第三步）

（1）create（创建）

Start with CentOS 7.0 to create a virtual machine for the Linux system (create four, one NameNode, and three DataNodes).

（先用CentOS7.0创建Linux系统的虚拟机（创建四台，一台NameNode和SecondaryNameNode，三台DataNode）。）

Then go straight to the next step, and finally start the virtual opportunity to install the Linux system automatically.Modify the network configuration after installation.Finally, you need to modify the network adapter for windows

（然后一直点下一步，最后启动虚拟机会自动安装Linux系统。安装完成之后修改一下网络配置。最后还要修改windows的网络适配器。）

After login to the virtual machine to modify the IP address, you need to modify the IP of the network adapter just set up under the same network segment (note: IP of four virtual machines,and then you need to modify the name of a virtual machine and the IP mapping.

(之后登陆虚拟机修改ip地址，需要修改成刚刚设置的windows的网络适配器的ip在同一网段下面（注：四台虚拟机的ip要在同一网段，但是ip要各不相同），然后还要修改一个虚拟机的名称以及ip映射。)

Finally, CRT is used to connect the virtual machine to see if the connection is successful.

（最后使用CRT连接虚拟机，看是否能够连接成功！）

（2）upload（上传）

After the connection is successful, the tar package downloaded from windows is uploaded to the virtual machine (note: only need to configure the NameNode virtual machine, the rest of the DataNode virtual machine can carry out remote data transmission through scp.Jdk’s tar package uploads the same as hadoop’s tar package.)

（连接成功之后便是将windows下载的tar包上传到虚拟机当中（注：只需要配置NameNode虚拟机，其余的DataNode虚拟机可以通过scp进行远程数据传输，jdk的tar包和hadoop的tar包一样的上传））

（3）configure（配置）

Configuration files need to be modified: hadoop-env.sh, core-site.xml, hdfs-site.xml, slaves, mapred-site.xml.template changed to mapred-site.xml, yarn-site.xml

（需要修改的配置文件：hadoop-env.sh、core-site.xml、hdfs-site.xml、slaves、mapred-site.xml.template修改为mapred-site.xml、yarn-site.xml）

NameNode formatting using [root@master hadoop]# hdfs namenode-format(Note: Format only the first time NameNode is started)

（使用[root@master hadoop]# hdfs namenode -format进行NameNode格式化（注：只有第一次启动NameNode时要格式化））

Start the Hadoop cluster at the NameNode node using [root@master hadoop]# start-all.sh.

（在NameNode节点启动Hadoop集群，使用[root@master hadoop]# start-all.sh ）

Finally, using JPS in each virtual machine, you can see how the Hadoop cluster works.

（最后在每个虚拟机中使用jps即可看到Hadoop集群的运行状况。）
