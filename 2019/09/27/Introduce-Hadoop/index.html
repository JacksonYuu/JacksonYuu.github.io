What is Hadoop？（什么是Hadoop?）

Hadoop is a distributed system infrastructure developed by the Apache Foundation. Implements a hadoop distributed file system (HDFS)。Simply speaking, it is a system used to store data, but the way of storage is different from the traditional way of storage. Traditional storage is to store all information in a single storage system, while distributed storage is to store information in different storage systems in a block way.

(Hadoop是一个由Apache基金会所开发的分布式基础架构。实现了一个分布式文件系统（Hadoop Distributed File System），简称HDFS。简单的来说就是用来存储数据的一个系统，只不过存储的方式和传统的存储方式不一样，传统的存储就是将所有的信息都存储在单一的存储体系当中，而分布式存储是将信息以块的方式分别存储在不同的存储体系当中。)

Advantages of Hadoop(Hadoop的优点)

1、High reliability（高可靠性）

Since multiple copies of working data are maintained when storing data, it is ensured that processing can be redistributed to failed nodes.

（由于在存储数据的时候会有维护多个工作数据副本，确保能够针对失败的节点重新分布处理。）

2、High scalability（高扩展性）

Since the stored data is stored on multiple machines in block form, the number of horizontally extended machines can store more data, which is much larger than the traditional storage on a single machine.

（由于存储数据是以块的形式存储在多个机器上面，横向扩展机器的个数便能够存储更多的数据，相对于传统的存储在单一机器上面来说扩展会更加的高效和方便。）

3、Efficiency（高效性）

Each node can dynamically transmit data, and ensure the dynamic balance of each node. Similarly, it can find the required data more quickly when responding to the client’s request.

（个个节点之间能够动态的传输数据，并且保证个个节点的动态平衡，同样在响应客户端的请求时能够更加迅速的找到所需的数据。）

4.High fault tolerance（高容错性）

Because there is redundant storage when storing data, and when the client requests to read the data, it can check the data. If the data error is detected, the backup data of the data block will be read, and the redundancy of the stored data will be kept to the set value.

（由于在存储数据的时候是具有冗余存储的，并且在客户端请求读取数据的时候具有对数据的检查，如果检查出数据错误，那么就会将这个数据块的备份数据进行读取，同时将存储数据的冗余数保持到设置的值。）

5、Low cost（低成本）

Compared with traditional storage, Hadoop can be a common PC for data storage machines. And Hadoop is an open source product that can be quickly used in the market.

（相对于传统存储来说，Hadoop对于数据存储的机器可以是普通的pc机。而且Hadoop是一种开源的产品，能够快速的运用在市场当中。）

Core components（核心组件）

1、NameNode

1、NameNode is a software that normally runs on a separate machine in an HDFS instance and has only one in a Hadoop cluster. It manages filesystem namespaces and controls access to external clients. NameNode decides whether to map a file to a replica block on DataNode. NameNode and DataNode have heartbeat connections. If NameNode does not receive the heartbeat of DataNode, NameNode stores the data of DataNode onto other DataNodes. Generally speaking, NameNode is the management and data preservation of DataNode.

（1、NameNode 是一个通常在 HDFS 实例中的单独机器上运行的软件，并且在一个Hadoop集群中有且只有一个。它负责管理文件系统名称空间和控制外部客户机的访问。NameNode 决定是否将文件映射到 DataNode 上的复制块上。NameNode和DataNode有着心跳连接，如果NameNode没有接收到DataNode的心跳，那么NameNode就会将该DataNode的数据存储到其他DataNode上去。笼统来说NameNode是对于DataNode进行管理和数据保存位置信息的管理。）

2、And two core data structures - FsImage and EditLog are saved. FsImage maintains metadata in all files and folders of the file system tree and the file tree species. EditLog records all additions and deletions to files. When NameNode starts, FsImage maps the number of filesystems into memory, and then recreates the new FsImage and EditLog after executing the records in EditLog. When NameNode runs, Secondary NameNode communicates regularly with NameNode, requesting it to stop writing to EditLog and create.

（并且保存了两个核心的数据结构-FsImage和EditLog。FsImage维护了文件系统树以及文件树种所有的文件和文件夹里的元数据。EditLog记录了所有对于文件的增删改的操作。在NameNode启动时，FsImage会将文件系统数映射在内存中，之后将EditLog里面的记录进行执行执行完成后会重新创建新的FsImage和EditLog。在NameNode运行时，SecondaryNameNode会定期和NameNode进行通信，要求其停止往EditLog中写入，创建一个新的文件edit.new来进行存放信息，同时SecondaryNameNode将NameNode的FsImage和EditLog下载到本机，在本机进行FsImage和EditLog的合并，合并完成后将FsImage重新发送给NameNode，这时NamenNode将FsImage进行替换，同时NameNode会将edit.new改成EditLog文件。）

2、DataNode

DataNode is also a software that normally runs on separate machines in HDFS instances. And it contains a large number of DataNodes in a Hadoop cluster. DataNode is responsible for the storage of data, and periodically sends a heartbeat to NameNode to prove its operation and report on the storage of information.

（DataNode 也是一个通常在 HDFS实例中的单独机器上运行的软件。并且在一个Hadoop 集群中包含大量 DataNode。DataNode负责的是对于数据的存储，同时会定期向NameNode发送心跳，来证明自己的运作和存储信息的报告。）

3、SecondaryNameNode

Communicate with NameNode and merge FsImage and EditLog. Similarly, when you can’t get NameNode’s heartbeat, you will convert yourself to NameNode. NameNode’s FsImage and EditLog will be downloaded locally and merged before they can take on the task of NameNode.

（和NameNode进行通信，合并FsImage和EditLog。同样在得不到NameNode的心跳时，会将自己向NameNode进行转换。会先将NameNode的FsImage和EditLog下载到本地，然后对其进行合并，合并之后才能够担当起NameNode的任务。）
