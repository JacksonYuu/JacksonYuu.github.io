What is zookeeper?（什么是Zookeeper？）

ZooKeeper is a distributed, open source distributed application coordination service, an open source implementation of Google’s Chubby, and an important component of Hadoop and Hbase. It is a software that provides consistency services for distributed applications. Its functions include configuration maintenance, domain name service, distributed synchronization, group service, etc.Simply put, it’s a small file storage system (storage system configuration information), which monitors the notification mechanism.

（ZooKeeper是一个分布式的，开放源码的分布式应用程序协调服务，是Google的Chubby一个开源的实现，是Hadoop和Hbase的重要组件。它是一个为分布式应用提供一致性服务的软件，提供的功能包括：配置维护、域名服务、分布式同步、组服务等。简单来说就是小文件存储系统（存储系统配置信息），监听通知机制）

Small File Storage System（小文件存储系统）

In Zookeeper, all directory nodes are called Znode, which can be divided into two types:

(在Zookeeper中所有的目录节点都叫做Znode，Znode又分为四种类型的节点：)

  • Persistent Nodes（持久性节点）

    In Zookeeper, as long as you don’t actively delete it, it will always exist.And it has uniqueness.

    （在Zookeeper中只要不主动删除就会一直存在）

  • Temporary Nodes（临时性节点）

    When a service disconnects from Zookeeper or goes down, the Znode is deleted on its own initiative.

    （当有服务与Zookeeper断开连接或者是宕机的时候，会主动将此Znode删除）

  • Ersistent Ordered Directory Nodes（持久性有序目录节点）

    In Zookeeper, as long as you don’t actively delete it, it will always exist.And it has uniqueness.

    （在Zookeeper中只要不主动删除就会一直存在，并且具有唯一性）

  • Temporary Ordered Directory Nodes（临时性有序目录节点）

    When a service disconnects from Zookeeper or goes down, the Znode is deleted on its own initiative.And it has uniqueness.

    （当有服务与Zookeeper断开连接或者是宕机的时候，会主动将此Znode删除，并且具有唯一性）

Monitoring Notification Mechanism（监听通知机制）

Zookeeper listens for directory nodes it cares about, and notifies clients when directory nodes change (data changes, deletions, subdirectory nodes are added and deleted).

（Zookeeper监听它关心的目录节点，当目录节点发生变化（数据改变、被删除、子目录节点增加删除）时，zookeeper会通知客户端。）

Electoral Mechanism（选举机制）

When Zookeeper’s main node (leader) goes down, a new leader is selected by the election mechanism inside Zookeeper (principle: a server initiates a polling request to the Zookeeper cluster, and more than half of the servers that get the most votes become leaders).

（当Zookeeper的主节点（leader）发生宕机的时候，Zookeeper内部的选举机制就会选出一个新的leader（原理：由一台服务器向Zookeeper集群发起投票请求，得票最多并且超过半数以上的服务器成为leader））
