What is HBase?（什么是HBase?）

HBase is a highly reliable, high-performance, column-oriented and scalable distributed database. It is an open source implementation of Google BigTable. It is mainly used to store unstructured and semi-structured loose data. The goal of HBase is to process very large tables, which can be scaled horizontally, using cheap computer clusters to process tables consisting of more than 1 billion rows of data and millions of columns of elements.

（HBase是一个高可靠、高性能、面向列、可伸缩的分布式数据库，是谷歌BigTable的开源实现，主要用来存储非结构化和半结构化的松散数据。HBase的目标是处理非常庞大的表，可以通过水平扩展的方式，利用廉价计算机集群处理由超过10亿行数据和数百万列元素组成的数据表）

Advantages of HBase（HBase的优点）

  1. Data Type（数据类型）

    Relational databases use relational models with rich data types and storage methods, while HBase uses a simpler data model, which stores data as unexplained strings.

    （关系数据库采用关系模型，具有丰富的数据类型和存储方式，HBase则采用了更加简单的数据模型，它把数据存储为未经解释的字符串）

  2. Data Manipulation（数据操作）

    Relational databases contain rich operations, which involve complex multi-table joins. HBase operation does not have complex table-to-table relationship, only simple insertion, query, deletion, empty, etc., because HBase design avoids complex table-to-table relationship.

    （关系数据库中包含了丰富的操作，其中会涉及复杂的多表连接。HBase操作则不存在复杂的表与表之间的关系，只有简单的插入、查询、删除、清空等，因为HBase在设计上就避免了复杂的表和表之间的关系）

  3. Storage Mode（存储模式）

    Relational databases are stored in row mode. HBase is based on column storage. Each column family is saved by several files. The files of different column families are separated.

    (关系数据库是基于行模式存储的。HBase是基于列存储的，每个列族都由几个文件保存，不同列族的文件是分离的)

  4. Data Index（数据索引）

    Relational databases can usually build complex multiple indexes for different columns to improve data access performance. HBase has only one index - row keys. By clever design, all access methods in HBase are accessed either by row keys or by row keys scanning, so that the whole system does not slow down.

    （关系数据库通常可以针对不同列构建复杂的多个索引，以提高数据访问性能。HBase只有一个索引——行键，通过巧妙的设计，HBase中的所有访问方法，或者通过行键访问，或者通过行键扫描，从而使得整个系统不会慢下来）

  5. Data Maintenance（数据维护）

    In a relational database, the update operation replaces the old value in the record with the latest current value, which does not exist after being overwritten. When updating is performed in HBase, the old version of data is not deleted, but a new version is generated, and the old version is still retained.

    （在关系数据库中，更新操作会用最新的当前值去替换记录中原来的旧值，旧值被覆盖后就不会存在。而在HBase中执行更新操作时，并不会删除数据旧的版本，而是生成一个新的版本，旧有的版本仍然保留）

  6. Scalability（可伸缩性）

    It is difficult for relational databases to expand horizontally, and the space for vertical expansion is limited. On the contrary, distributed databases such as HBase and BigTable are developed to achieve flexible horizontal scaling, which can easily scale performance by increasing or reducing the number of hardware in the cluster.

    （关系数据库很难实现横向扩展，纵向扩展的空间也比较有限。相反，HBase和BigTable这些分布式数据库就是为了实现灵活的水平扩展而开发的，能够轻易地通过在集群中增加或者减少硬件数量来实现性能的伸缩）

HBase Data Model（HBase数据模型）

  1. HBase is a sparse, multi-dimensional, sorted mapping table whose indexes are row keys, column families, column qualifiers, and timestamps.

    （HBase是一个稀疏、多维度、排序的映射表，这张表的索引是行键、列族、列限定符和时间戳 ）

  2. Each value is an unexplained string with no data type

    （每个值是一个未经解释的字符串，没有数据类型）

  3. Users store data in tables, each row has a sortable row key and any number of lists in the horizontal direction consisting of one or more column families, a column family can contain any number of columns, and the data in the same column family can be stored together.

    （用户在表中存储数据，每一行都有一个可排序的行键和任意多的列表在水平方向由一个或者多个列族组成，一个列族中可以包含任意多个列，同一个列族里面的数据存储在一起 ）

  4. Column family supports dynamic expansion. It is easy to add a column family or column without pre-defining the number and type of columns. All columns are stored in the form of strings. Users need to convert data type by themselves.

    （列族支持动态扩展，可以很轻松地添加一个列族或列，无需预先定义列的数量以及类型，所有列均以字符串形式存储，用户需要自行进行数据类型转换）

  5. When updating is performed in HBase, the old version of data is not deleted, but a new version is generated, and the old version is still retained (this is related to the feature that HDFS only allows additional and no modification)

    （HBase中执行更新操作时，并不会删除数据旧的版本，而是生成一个新的版本，旧有的版本仍然保留（这是和HDFS只允许追加不允许修改的特性相关的））
