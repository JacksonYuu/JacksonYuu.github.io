What is MapReduce?（什么是MapReduce？）

  • MapReduce is a distributed parallel programming model for parallel computing of large data sets (larger than 1TB)

    （MapReduce是一种分布式并行编程模型，用于大规模数据集（大于1TB）的并行运算）

  • MapReduce highly abstracts the complicated parallel computing process running on large-scale cluster into two functions: Map and Reduce, which makes programming easy, and does not need to master the details of distributed parallel programming. It is also easy to run its own programs on distributed systems to complete the calculation of massive data.

    （MapReduce将复杂的、运行于大规模集群上的并行计算过程高度地抽象到了两个函数：Map和Reduce，这样就使得编程容易，不需要掌握分布式并行编程细节，也可以很容易把自己的程序运行在分布式系统上，完成海量数据的计算）

  • MapReduce uses a “divide and conquer” strategy. A large-scale data set stored in a distributed file system is divided into many separate fragments, which can be processed in parallel by multiple Map tasks.

    （MapReduce采用“分而治之”策略，一个存储在分布式文件系统中的大规模数据集，会被切分成许多独立的分片（split），这些分片可以被多个Map任务并行处理）

  • One of the ideas of MapReduce design is that “computing is closer to data” rather than “data is closer to computing”, because mobile data requires a lot of network transmission overhead.”Computing to Data Close” is the node where the task is assigned to the data, and “Data to Computing Close” is the node where the data is transmitted from other nodes to the running task.

    （MapReduce设计的一个理念就是“计算向数据靠拢”，而不是“数据向计算靠拢”，因为，移动数据需要大量的网络传输开销。“计算向数据靠拢”是任务被分配到数据所在的节点上，“数据向计算靠拢”是数据从其他节点传输到运行任务的节点上）

  • MapReduce framework uses Master/Slave architecture, including one Master and several Slaves. Hadoop version 1.0 is JobTracker running on Master, TaskTracker running on Slave, and Resource Coordinator YARN added to Hadoop version 2.0, ResourceManager running on Master and NodeManager running on Slave.

    （MapReduce框架采用了Master/Slave架构，包括一个Master和若干个Slave。Hadoop1.0的版本是Master上运行JobTracker，Slave上运行TaskTracker，Hadoop2.0版本添加了YARN这个资源协调者之后，Master上运行ResourceManager，Slave上运行NodeManager）

Components of MapReduce（MapReduce的组成）

  • Client

    MapReduce programs written by users are submitted to JobTracker through Client, and the running status of jobs can be viewed through some interfaces provided by Client.

    （用户编写的MapReduce程序通过Client提交到JobTracker端，同时可通过Client提供的一些接口查看作业运行状态）

  • JobTracker（ResourceManager）

    1. Responsible for resource monitoring and job scheduling, while monitoring the health of all TaskTracker and Job, once the failure is found, the corresponding tasks will be transferred to other nodes.

      （负责资源监控和作业调度，同时监控所有TaskTracker与Job的健康状况，一旦发现失败，就将相应的任务转移到其他节点。）

    2. Tracking the progress of task execution, resource usage and other information, and informing the task scheduler (Task Scheduler, in 2.0, YARN), and the scheduler will select the appropriate tasks to use these resources when resources are free.

      （跟踪任务的执行进度、资源使用量等信息，并将这些信息告诉任务调度器（TaskScheduler，在2.0中为YARN），而调度器会在资源出现空闲时，选择合适的任务去使用这些资源）

  • TaskTracker（NodeManager）

    1. Periodically report the resource usage and task running progress to JobTracker through heartbeat. At the same time, receive the commands sent by JobTracker and perform the corresponding operations (such as starting new tasks, killing tasks, etc.).

      （周期性地通过“心跳”将本节点上资源的使用情况和任务的运行进度汇报给JobTracker，同时接收JobTracker 发送过来的命令并执行相应的操作（如启动新任务、杀死任务等））

    2. The resource amount (CPU, memory, etc.) on this node is divided by “slot” equivalence. A Task gets a slot before it has a chance to run, and the function of the Hadoop scheduler is to allocate the free slots on each Task Tracker to Task. Slot is divided into Map slot and Reduce slot, which are used by MapTask and Reduce Task respectively.

      （使用“slot”等量划分本节点上的资源量（CPU、内存等）。一个Task 获取到一个slot 后才有机会运行，而Hadoop调度器的作用就是将各个TaskTracker上的空闲slot分配给Task使用。slot 分为Map slot 和Reduce slot 两种，分别供MapTask 和Reduce Task 使用）

  • Task

    Task is divided into Map Task and Reduce Task, both started by TaskTracker.

    （Task 分为Map Task 和Reduce Task 两种，均由TaskTracker 启动）
