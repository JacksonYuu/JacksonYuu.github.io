What is Kafka？（什么是Kafka？）

Kafka is a high throughput, distributed publish and subscribe message system based on the coordinated management of zookeeper. It can process all the action flow data of consumers in the website. Kafka aims to unify online and offline message processing through Hadoop’s parallel loading mechanism, and also to provide real-time messages through clusters.

（Kafka是一种高吞吐量的，是基于zookeeper协调管理的分布式发布订阅消息系统，它可以处理消费者在网站中的所有动作流数据。Kafka的目的是通过Hadoop的并行加载机制来统一线上和离线的消息处理，也是为了通过集群来提供实时的消息。）

Characteristics of Kafka（Kafka的特性）

  • Through time complexity, message persistence is provided for O (1) disk data structure, and even TB message storage can maintain long-term stable performance.

    （通过时间复杂度为O(1)的磁盘数据结构提供消息的持久化，即使是TB的消息存储也能够保持长时间的稳定性能。）

  • High throughput: even very common hardware Kafka can support millions of messages per second.

    （高吞吐量 ：即使是非常普通的硬件Kafka也可以支持每秒数百万的消息。）

  • It supports message partition through Kafka server and consumer cluster.

    （支持通过Kafka服务器和消费机集群来分区消息。）

  • Supports Hadoop parallel data loading.

    （支持Hadoop并行数据加载。）

What is publish subscribe message system？

（什么是发布订阅消息系统？）

This system includes the information publisher and the information consumer. The message publisher publishes the message, and the message is persisted to a topic. The message consumer can subscribe to one or more topics to read the message, and the message will not be deleted after reading.All message consumers who have subscribed to the topic will not delete the message until they successfully read the message.

（这个系统中包括消息发布者，消息消费者。消息发布者进行消息的发布，消息被持久化到一个topic中，消息消费者可以订阅一个或多个topic读取消息，消息并不会在读取之后进行删除。所有订阅了该topic的消息消费者成功读取消息后，消息才会进行删除。）

Components of Kafka（Kafka的组件）

  • Broker

    Servers in a Kafka cluster are called brokers.

    （在Kafka集群中的服务器都被称为broker）

  • Topic

    Each message published to the Kafka cluster has a category called topic. The messages above the topic will be partitioned to achieve load balancing.

    （每条发布到Kafka集群的消息都有一个类别，这个类别被称为Topic。Topic上面的消息会被进行分区，从而达到负载均衡）

  • Partition

    The messages above the topic will be partitioned, and the partitioned block will become partition.Each partition has multiple copies, among which there is only one as the leader, who is currently responsible for reading and writing data.

    （Topic上面的消息会被进行分区，分区后的块成为partition。每个partition有多个副本，其中有且仅有一个作为Leader，Leader是当前负责数据的读写的partition。）

  • Producer

    Responsible for publishing messages to the broker in the Kafka cluster through the TCP protocol.

    （负责发布消息到Kafka集群中的broker，通过TCP协议。）

  • Consumer

    Message consumer, the client that reads the message to the broker in the Kafka cluster. It also transmits data through TCP protocol.

    （消息消费者，向Kafka集群中的broker读取消息的客户端。也是通过TCP协议进行数据的传输。）
